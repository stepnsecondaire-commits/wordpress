## 6. Security: the 2 secrets in wp_options

We identified the exact location of the two secrets GitHub secret scanning caught during Phase 1 push:

1. **OpenAI API key** is stored in the WordPress option `breakdance_api_keys`. This is the OpenAI key used by the Breakdance builder's "AI Assist" feature for copy suggestions in the visual editor. Deleting it would only break that feature (admin-only). If Ryan is not using the AI Assist, delete this key from Breakdance settings: go to `WP Admin > Breakdance > Settings > AI`, clear the OpenAI key field, save. Then revoke the key itself on https://platform.openai.com/api-keys.

2. **Google OAuth access token** is stored in the WordPress option `surerank_google_console_credentials`. This is the active OAuth session that allows SureRank to pull Google Search Console data. **Deleting this would break our GSC integration.** The right action here is to **periodically disconnect and reconnect** from `WP Admin > SureRank > Search Console > Disconnect`, which invalidates the old token and issues a fresh one. Every 3 to 6 months is a reasonable rotation cadence.

### General hardening reminders

- Enable 2-factor authentication on the single admin account (`eyvenbest@163.com`).
- Rotate the WordPress admin password every 90 days.
- Enable database backup encryption on Hostinger (Control Panel > Files > Backups > enable encryption).
- Never commit the database dump or the `wp-config.php` file to any git repository. We already added them to `.gitignore` in our project.
