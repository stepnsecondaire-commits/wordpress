# blog-auto

Automated blog publishing pipeline for eviehometech.com.

## How it works

1. `articles.json` holds the full schedule of 100 articles organized in 8 topical clusters, each with a scheduled publication datetime.
2. `publish.py` runs on a GitHub Actions cron every 30 minutes. Each run picks the next unpublished article whose scheduled datetime has passed, generates the article body (either from a pre-written HTML file in `content/` or via Claude Sonnet 4), fetches a relevant Unsplash feature image, publishes the post to WordPress via REST, sets the SureRank SEO meta, pings IndexNow to notify Bing and Yandex, and marks the article as published in `articles.json`.
3. `logs.txt` records every run.

## Publishing schedule

Articles are scheduled over 6 weeks (2026-04-15 to 2026-06-01), 2 to 3 per business day, Tuesday to Thursday are the heaviest days. Publishing slots:
- 06:00 UTC (7h Paris / 8h Berlin / 1h EST / previous day 22h PST) to catch Europe morning
- 14:00 UTC (15h Paris / 9h EST / 6h PST) to catch US East Coast business hours
- 18:00 UTC (19h Paris / 13h EST / 10h PST) to catch US West Coast business hours

## Required GitHub Actions secrets

| Secret | Purpose |
|---|---|
| `ANTHROPIC_API_KEY` | Claude Sonnet 4 API key to generate the 92 satellite articles |
| `WP_BASIC_AUTH` | Base64 `user:app_password` for WordPress REST Basic Auth |
| `UNSPLASH_ACCESS_KEY` | Unsplash API for feature images (optional, uses a default shared key if missing) |
| `INDEXNOW_KEY` | IndexNow key for Bing and Yandex ping (default: `eviehome-indexnow-7f3a2b8c9d1e4f5a`) |

The `WP_BASIC_AUTH` secret must be generated from a WordPress Application Password:
1. Log in to `/wp-admin/`
2. Go to Users > Your Profile
3. Scroll to Application Passwords section
4. Create a new app password named "blog-auto"
5. Format it as `username:app_password` and base64-encode: `echo -n "username:app_password" | base64`
6. Paste the result into the GitHub secret `WP_BASIC_AUTH`

## Local testing

```bash
# Dry run (no publication)
python3 blog-auto/publish.py --dry-run

# Force publish a specific article index (bypasses the schedule)
ANTHROPIC_API_KEY=xxx WP_BASIC_AUTH=yyy python3 blog-auto/publish.py --force 42
```

## Updating the schedule

Edit `articles.json` to change scheduled datetimes or add new articles. Commit and push. The next cron run will pick up the changes.

## Adding a pre-written pillar article

1. Write the HTML in `content/NNN-slug.html` where `NNN` is the three-digit index and `slug` is the article slug from articles.json.
2. Keep the first `<p>` paragraph strong because it is used as the meta description fallback if Claude is not called.
3. Commit and push. On the next cron run, `publish.py` detects the file and uses it instead of generating content.
